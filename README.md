
# 対象者

- インフラエンジニア
  - IaCの設計を担当する
  - IaC初心者でIaC導入に向けて何を考えたらいいかわからない

# 利用者に与える価値

- IaCの経験がなくても開発・運用しやすいIaCが設計できる
- 要件に沿った設計か確認できる
- 設計の中には選択肢のあるものもある。利用者は好みに応じて設計を選択できる
- 要件・設計をあとから変更できる

# 利用の流れ

1. 利用者はWEB経由でサービスにアクセスする
2. サービス開始画面には**要件**と**設計**の画面に移動できる
3. 利用者はまず**要件**を選択する
   1. **要件**では一般的にIaCに求められる**機能要件**とIaCに関連した**非機能要件**が入力できる
   2. **機能要件**および**非機能要件**はサービスが要件レベルを提示する。利用者は要件レベルを選択する。選択時に理由をテキストで入力できる
   3. 入力が終わった**要件**はファイルとして利用者はDLできる
   4. DLした要件ファイルをエクスポートすると、利用者は何度でも**要件**の状態を復元できる
4. **要件**を入力したら**設計**を選択する
   1. **設計**ではIaCに関する設計が入力できる
   2. 各設計ではその設計についての説明と設計パターンを提示する
   3. 提示されたパターンから利用者は設計を選択できる。また、選択する時に理由をテキストで入力できる。
   4. 選択パターンのデフォルトはサービスが要件を読み取った結果、最適だと判断したパターンが選択されている。
   5. もし選択したパターンだと要件を満たさなくなる場合、サービスは利用者に警告を伝える
   6. 入力が終わった**設計**はファイルとして利用者はDLできる
   7. DLした設計ファイルをエクスポートすると、利用者は何度でも**設計**の状態を復元できる

# 追加機能のアイディア

- **要件**はよくある3パターン(Poc用、社内サービス用、公開サービス用)などから選べるようにする
- 利用者のGitにある要件ファイルと設計ファイルを読み込む
- **設計**からIaCの雛形コード生成
- 仕組みを流用してIaC以外の設計も行えるようにする

# タスク

1. 要件機能
  1. 要件およびレベルの雛形作成
  2. 雛形を基にWEB画面で要件とレベルの提示
  3. 利用者で要件を選択できるようにし、同時に理由もテキストで入力できるようにする
  4. 要件をファイルでエクスポート/インポートできるようにする
2. 設計機能
  1. 設計および設計パターンの雛形作成
  2. 雛形を基にWEB画面で設計とパターンを提示
  3. 利用者で設計を選択できるようにし、同時に理由もテキストで入力できるようにする
  4. 設計をファイルでエクスポート/インポートできるようにする
  5. 要件を読み取り、設計パターンのデフォルトを選択するようにする(ルールベース or ML)
  6. 選択した設計が要件に合わない場合は警告を表示する
